SHARP Smart Office Assistant

Fall 2016 - Winter 2017

A proof of concept Android application designed to facilitate and enhance the productivity of meetings for organizations and groups.

See demo of the application in InVision under "View Demo" link below. Actual application was created in native Android Studio and is functional on an actual Android device.

Role: Lead front-end designer that developed 95% of the user interface and connected the front-end with the back-end via API calls. Also worked on wireframes, UML diagram, test cases, scenarios, personas, user testing, user requirements and business case and software and UI design.

Deliverables: A functional proof of concept Android application (for phone and Tablet), Project Plan, User Requirements, Business Case, Software and UI Design document, Test Cases, Test Plans, Usability Evaluations, User Tests and Results, and a full documentation of suggestions and changes we would have liked to make had we been given more time.

About: The Smart Office Assistant application was designed and developed under our sponsors at SHARP Labs of America for my senior capstone project over the span of 6 months (INF 191 series at UC Irvine) in a group of 5. There are 2 components of the Smart Office Assistant: manual meeting creation and display and creating a meeting via Chatbot, which displays the meeting on a Google Calendar.

The app has a Home screen that is intended to display the list of meetings created with the title, date, time and location of the meeting. On the Home Screen, the user can either choose to create a meeting manually or tap "Chatbot" to converse with an interactive AI to create a meeting. Clicking each meeting object on the Home screen will direct the user to the specific meeting's Timeline which displays the list of topics for that meeting and a timer associated with each topic. Each topic is displayed in an expandable/collapsible dropdown menu where expanding it will display more information and allow a place to take and save notes under it.

The interactive Chatbot works by allowing the user to enter a specific set of commands and the Chatbot will automatically respond. The user can use ChatBot to create a meeting with a set date, time and location or update, delete or extend a meeting. Their changes are then scheduled and displayed via Google Calendars.

Working Functionalities: The proof of concept app is capable of displaying all meeting information in one place, creating, deleting, and updating meetings and topics for each meeting and creating, updating or deleting meetings automatically via an interactive Chatbot.

Personal Website From Scratch

A fully responsive basic personal website created from scratch using HTML, CSS, and Javascript for my User Interaction Software class.

This website was created without Bootstrap or any templates. For this project, user research and user testing was also completed with a full analysis of changes that I would have liked to have implemented.

Click "View Site" at the bottom to find the full report and test results under the "Final Project" tab.

I used the following methods on this project:

Method: User Research

  • User Interviews - to better understand the possible users and their different expectations from the product.
  • Competitor Analysis - to evaluate the features, strengths and weaknesses of my website in comparison to others' website in my field.
  • Feature Value Matrix & Feature Prioritization - to rank the features that were implemented and what features should be implemented in order of value based off complexity and importance.

Method: User Testing

  • Cognitive Walkthrough - to evaluate the effectiveness and success of the website by giving a set of tasks to user testers to complete/answer.
  • User Stories - to explain the reasoning behind existing features and features that I wanted to implement that will provide a better user interface and experience.
  • User Test - Used a click test and a 5 second test to gauge what the first impression of the website is.
